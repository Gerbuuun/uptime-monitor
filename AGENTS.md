# Repository workflow

- Never commit or push directly to `main`. Create a `codex/<short-description>` branch for every cohesive feature, fix, chore, or documentation change.
- Open one draft pull request per branch and merge changes through that pull request. Base branches on `main` unless a stacked change explicitly depends on another open pull request.
- Use Conventional Commit messages. Add `!` only when the commit changes a public contract in a breaking way; do not use it for additive features, fixes, chores, or internal refactors.
- Do not modify, comment on, convert, close, merge, or otherwise interact with pull requests automatically generated by `uppt`.
- Before opening a pull request, run the relevant checks and include the change summary, user impact, and validation in the pull request body.
